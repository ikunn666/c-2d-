# NARUTO - 本地双人格斗游戏

基于 C++ 和 raylib 图形库开发的本地双人 2D 格斗游戏。项目采用面向对象设计，实现了状态机控制、碰撞检测以及火影忍者主题的角色技能。

项目开发全流程采用 Git 进行版本控制，保持了良好的提交历史与代码规范。

---

## 游戏操作说明

游戏采用单键盘双人对战模式，操作布局如下：

### 玩家 1（P1 - 左侧）

- 移动：A（向左）/ D（向右）
- 跳跃：K
- 普通攻击：J
- 技能一：U
- 技能二：I
- 替身术：L

### 玩家 2（P2 - 右侧小键盘）

- 移动：←（左箭头）/ →（右箭头）
- 跳跃：2（小键盘）
- 普通攻击：1（小键盘）
- 技能一：4（小键盘）
- 技能二：5（小键盘）
- 替身术：3（小键盘）

### 全局控制

- 胜负判定：任意一方生命值归零或时间60s倒计时结束时游戏立刻结束。
- 重新开始：游戏结束后，按下键盘上的 R 键可直接重启对局。

## 编译与运行指南

本项目基于 `raylib-quickstart` 模板构建，配置过程分为“环境初始化”与“编译调试”两个阶段。请严格按照以下顺序操作，否则可能导致构建失败。

### 第一阶段：初始化构建环境（仅需执行一次）

由于 raylib 源码在本地尚未下载完整，必须先运行构建脚本以补齐依赖：

1. **执行下载脚本**：在项目根目录下，运行 `build-MinGW-W64.bat`。

- 该脚本会自动从 GitHub 下载 raylib 源代码，并自动调用 `premake5` 工具生成项目所需的 `Makefile` 与施工图。
- _注意：若此时出现“异常行终止符”提示，请点击“忽略”。_

2. **确认配置成功**：脚本执行完毕后，检查根目录下是否出现了 `Makefile` 以及以 `.make` 结尾的配置文件。

### 第二阶段：编译与调试运行

环境准备就绪后，你可以通过以下任一方式启动游戏：

#### 选项 A：VS Code 一键编译调试 (推荐)

配置好环境后，直接使用 VS Code 的调试功能即可：

1. 打开 `src/main.cpp` 或工程根目录。
2. 直接按下键盘 **F5** 键。
3. VS Code 将自动调用预设的 `tasks.json` 进行编译，并在构建完成后启动调试器运行游戏。

#### 选项 B：命令行手动编译

如果你偏好使用终端进行管理：

1. 在项目根目录下打开终端。
2. 输入以下命令进行编译：

```bash
mingw32-make

```

3. 编译完成后，根目录下会生成可执行文件（如 `Naruto.exe`），在终端直接输入程序名即可启动。

---

**⚠️ 常见问题排查：**

- **编译报错/找不到头文件**：请确保已经完整执行过第一阶段的 `.bat` 脚本。如果网络不稳定导致下载中断，请删除 `build` 文件夹下的 `external` 内容后，配置代理重新运行 `.bat` 脚本。
- **F5 无法触发编译**：若按下 F5 报错，请检查 `.vscode/tasks.json` 是否正确配置，并确保终端当前路径位于项目根目录下。
